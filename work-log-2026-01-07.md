## 🎯 **Objective**
Transform SessionLauncher to use git bead chains as PRIMARY context source instead of database queries. Enable time-travel replay and collaborative workflows.

## 📋 **Architecture Decision**

### **Approach 1: Git-Primary (Recommended)**
- **Input:** SessionLauncher receives `lastBeadId` from git index
- **Process:** Read bead chain from git, reconstruct context
- **Output:** Injected prompt with git-traced memory
- **Result:** Agent sees EXACT same context as historical session

### **Approach 2: Hybrid (Fallback)**
- **Input:** Mix git beads + DB vectors
- **Process:** Git for context, Vector for similarity search
- **More complex but handles edge cases**

**Choice:** Approach 1 (Git-Primary) - cleaner, aligns with Ironclad philosophy
