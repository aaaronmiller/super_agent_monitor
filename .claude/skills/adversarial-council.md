# Adversarial Council Protocol

Initiate rigorous validation through simulated expert discourse.

## Execution

1. **Instantiate 10 Expert Personas** relevant to domain:
   - Security Auditor, UX Architect, Skeptic, Senior Engineer
   - Performance Expert, Edge Case Hunter, API Designer
   - Documentation Critic, DevOps Lead, End User Advocate

2. **Conduct 3 Iterative Rounds**:
   - Round 1: Initial critique from each persona
   - Round 2: Cross-examination of critiques
   - Round 3: Synthesis and consensus building

3. **Mandatory Grounding**:
   - Web search BEFORE Round 1 for context
   - Web search BETWEEN rounds to verify claims
   - No ungrounded assertions allowed

4. **Output Requirements**:
   - Conflict Map: Areas of disagreement
   - Consensus Action Plan: Agreed next steps
   - Dissenting Opinions: Unresolved concerns

## Anti-Patterns
- Do NOT just summarize
- Do NOT skip grounding searches
- Do NOT present false consensus
